# 前端项目使用说明

## 快速开始

### 1. 安装依赖

```bash
cd frontend
npm install
```

### 2. 启动后端服务

确保后端 Flask 应用已启动：

```bash
# 在项目根目录
python run.py
```

后端服务默认运行在 `http://localhost:5000`

### 3. 启动前端开发服务器

```bash
cd frontend
npm run dev
```

前端服务将在 `http://localhost:3000` 启动

## API 代理配置

前端通过 Vite 的 proxy 功能代理 API 请求：

- 前端访问: `http://localhost:3000/api/v1/laboratories/`
- 实际请求: `http://localhost:5000/api/v1/laboratories/`

配置位置：`vite.config.js`

## 项目结构说明

```
frontend/
├── src/
│   ├── api/              # API 请求封装
│   │   ├── request.js    # Axios 实例和拦截器
│   │   └── laboratory.js # 实验室相关 API
│   ├── router/           # 路由配置
│   ├── views/            # 页面组件
│   ├── App.vue           # 根组件（包含导航）
│   └── main.js           # 入口文件
├── index.html            # HTML 模板
├── package.json          # 依赖管理
└── vite.config.js        # Vite 配置
```

## 开发注意事项

1. **API 响应格式**: 后端统一返回 `{code, msg, data}` 格式，前端已在 `request.js` 中处理
2. **错误处理**: 所有 API 错误会自动通过 ElMessage 提示用户
3. **表单验证**: 使用 Element Plus 的表单验证规则
4. **加载状态**: 表格加载时显示 loading 状态

## 构建部署

```bash
npm run build
```

构建产物在 `dist/` 目录，可以部署到任何静态文件服务器。
